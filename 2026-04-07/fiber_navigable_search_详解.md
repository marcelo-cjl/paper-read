# Fiber-Navigable Search: A Geometric Approach to Filtered ANN

**arXiv ID**: 2604.00102  
**提交日期**: 2026-03-31 (2026-04-01 公布)  
**作者**: Thuong Dang  
**领域**: 过滤近似最近邻搜索 (Filtered ANN Search)

---

## 整体概览

```
问题：带元数据过滤的向量检索
+-----------------------------+
|  向量库 + 元数据属性        |
|  查询：找出满足条件的k近邻  |
+-----------------------------+
         |
         v
传统方法的问题：
+-----------------------------+
|  过滤后的子图(Fiber)连通性  |
|  与原图差异巨大             |
|  导致搜索失败               |
+-----------------------------+
         |
         v
本文方案：Fiber-Navigable Search
+-----------------------------+
|  几何框架分析失败模式       |
|  两阶段搜索 + Anchor结构    |
|  重启搜索机制               |
+-----------------------------+
```

Fiber-Navigable Search 提出了一套**几何学框架**来分析和解决过滤近似最近邻搜索（Filtered ANN）中的核心问题：当对图索引按元数据谓词过滤后，产生的子图（称为"fiber"）其连通性和几何特性与原图差异显著，导致搜索质量严重下降。

---

## 背景与动机

### 过滤ANN搜索的挑战

```
完整图 G                    过滤子图 Fiber(f)
+--[v1]--[v2]--+           +--[v1]----+
|   |      |   |    过滤    |          |
|  [v3]--[v4]  |   ====>   |   [v3]   |
|   |      |   |    f(v)   |          |
+--[v5]--[v6]--+           +--[v5]----+

完整图：高连通，搜索路径丰富
过滤子图：连通性差，可能出现孤立区域
```

**核心问题**：在图索引（如HNSW）中进行过滤搜索时，按谓词过滤后的局部图（fiber）可能：
1. 在查询附近缺乏连通性（**拓扑切割**）
2. 图节点空间分布扭曲（**几何折叠**）
3. 存在局部最优陷阱（**真实盆地**）

### 现有方法的不足

| 方法 | 策略 | 问题 |
|------|------|------|
| 后过滤 (Post-filtering) | 先搜索再过滤 | 低选择率时结果不足 |
| 预过滤 (Pre-filtering) | 只搜索满足条件的节点 | 破坏图连通性 |
| ACORN/HNSW改进 | 扩大搜索范围 | 代价高昂，无根本解决 |

## 核心方法：Fiber-Navigable Search

### Phase 1: 搜索失败模式分类

```
失败模式分类框架：

     查询 q
       |
       v
+------+------+
|  局部几何   |
|  信号检测   |
+------+------+
       |
   +---+---+
   |       |
   v       v
拓扑切割  几何折叠  真实盆地
(Topo.  (Geo.    (Genuine
 Cuts)   Folds)   Basins)
   |       |         |
   +---+---+---------+
       |
       v
  统一解决方案：
  重启至 fiber-present 簇
```

**三种失败模式**：

1. **拓扑切割 (Topological Cuts)**  
   过滤后图在查询区域彻底断连，greedy search 无法到达真实近邻

2. **几何折叠 (Geometric Folds)**  
   过滤条件导致向量空间几何变形，图的距离排序与实际不符

3. **真实盆地 (Genuine Basins)**  
   过滤后出现局部最优，搜索陷入非全局最优区域

**关键发现**：三种失败模式共享统一的解决方案——**在查询附近的 fiber-present 簇中重启搜索**。

### Phase 2: 两阶段搜索算法

```
阶段一：完整图探索
+------------------------+
| 在完整图 G 上运行      |
| greedy 搜索算法        |
| 收集候选节点集合        |
+------------------------+
         |
         | 局部几何良好? (fiber充足)
         |
    YES  |  NO
     v       v
阶段二a：      阶段二b：
过滤邻居下降   Anchor重启
+----------+  +----------+
| 在过滤   |  | 找fiber  |
| 子图上   |  | present  |
| 继续搜索 |  | 的anchor |
+----------+  +----------+
               |
               v
          重启搜索
```

**两阶段搜索逻辑**：
```
function FiberNavigableSearch(G, q, predicate f, k):
    # Phase 1: 完整图探索
    candidates = GreedySearch(G, q, ef)
    
    # 检测局部几何
    if LocalFiberGeometryFavorable(candidates, f):
        # Phase 2a: 过滤邻居下降
        return FilteredNeighborDescent(candidates, f, k)
    else:
        # Phase 2b: Anchor重启
        anchor = FindFiberPresentAnchor(q, f)
        return FiberNavigableSearch(G, anchor, f, k)
```

### Anchor 结构

```
Anchor 结构设计：
+-----------------------------------+
|  离线预处理：                     |
|  对数据集按属性值分区             |
|  每个分区保留代表性"锚点"         |
|                                   |
|  anchor[attr_val] = cluster_repr  |
+-----------------------------------+
         |
         v 查询时
+-----------------------------------+
|  1. 确定查询的谓词 f              |
|  2. 找到满足 f 的最近 anchor      |
|  3. 从 anchor 重启 fiber 搜索     |
+-----------------------------------+

空间代价：O(|A|) ≈ O(√n)  (轻量级)
```

**Anchor 结构特点**：
- 轻量级额外存储（约 O(√n)）
- 标识满足过滤条件的聚类中心（fiber-present 区域）
- 用于在搜索失败时提供可靠的重启点

## 搜索失败检测机制

```
失败检测信号：

当前搜索状态
     |
+----+----+
|         |
v         v
过滤率   邻居
检查     密度
|         |
低选择率  稀疏fiber
    |         |
    +----+----+
         |
         v
     判定失败模式:
     拓扑/几何/盆地
         |
         v
     触发 Anchor 重启
```

三种失败模式在**局部信号**上可区分：
- 过滤选择率（selectivity）增大时，三种模式按规律演变
- 可通过轻量级本地检测（无需全局信息）识别失败状态

## 实验结果

### 与 FAISS HNSW 对比

```
Recall@10 对比 (低选择率 ~1%):
方法            Recall   QPS
---------       ------   ----
FAISS HNSW      0.42     ████████ (baseline)
Fiber-Nav.      0.87     ███████  (+107% recall)

高选择率 ~50%:
方法            Recall   QPS
---------       ------   ----
FAISS HNSW      0.91     ████████████
Fiber-Nav.      0.95     ██████████   (+4% recall)
```

**关键实验结论**：
1. 低选择率（严格过滤条件）时，Fiber-Navigable Search 召回率提升最显著
2. 三种失败模式随过滤选择率变化而有规律地转变
3. Anchor 结构增加的存储开销可忽略不计

## 总结

**Fiber-Navigable Search 的核心贡献**：

1. **理论框架**：首次将过滤ANN搜索失败归类为三种几何模式（拓扑切割/几何折叠/真实盆地）

2. **算法设计**：两阶段搜索（完整图探索 + fiber感知下降/重启）无缝结合

3. **轻量结构**：Anchor 结构以 O(√n) 额外存储支撑重启机制

4. **实用效果**：低选择率下对 FAISS HNSW 召回率提升显著（约+100%）

**适用场景**：
- RAG系统中带元数据过滤的语义搜索
- 电商/推荐系统中多条件约束的向量检索
- 任何需要精确过滤条件的高维相似搜索

**局限性**：
- 主要针对图索引（HNSW等）设计
- IVF类索引需要不同的处理框架
- Anchor的离线预处理需要了解过滤谓词分布
