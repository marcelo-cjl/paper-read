## 背景与动机

### 要解决什么问题？

大语言模型在推理时需要维护 **KV 缓存**（Key-Value Cache），随着上下文长度增加，KV 缓存占用的内存急剧膨胀。传统量化方法存在一个根本性矛盾：

```
┌─────────────────────────────────────────────────────────────┐
│                    传统量化的矛盾                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  量化目标: 将 32-bit 浮点数压缩到 3-4 bit                  │
│                                                             │
│  但是! 每个数据块需要存储量化常数:                          │
│                                                             │
│  ┌────────────────┐    ┌────────────────┐                  │
│  │  量化后的数据   │ +  │  量化常数(全精度)│                  │
│  │  3-4 bits/维   │    │  1-2 bits/维    │                  │
│  └────────────────┘    └────────────────┘                  │
│          │                      │                           │
│          └──────────┬───────────┘                           │
│                     ▼                                       │
│            实际压缩率大打折扣!                               │
│            "量化常数的开销抵消了量化的收益"                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 为什么现有方法不够好？

```
┌─────────────────────────────────────────────────────────────┐
│                   现有方法的局限                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  KIVI 等方法:                                               │
│  ┌─────────────────────────────────┐                       │
│  │  需要数据标准化 (归一化)         │                       │
│  │  量化常数占用额外内存            │                       │
│  │  需要数据集特定调优              │                       │
│  └─────────────────────────────────┘                       │
│                                                             │
│  乘积量化 (PQ):                                             │
│  ┌─────────────────────────────────┐                       │
│  │  需要大型码本                    │                       │
│  │  需要训练/调优                   │                       │
│  │  不适合在线场景                  │                       │
│  └─────────────────────────────────┘                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Phase 1: PolarQuant — 极坐标量化

### 1.1 目的

将高维向量从笛卡尔坐标系转换到极坐标系，利用角度分布的集中性实现高效量化，**无需数据标准化**。

### 1.2 核心思想

```
直觉理解:

笛卡尔坐标 (传统方式):
┌─────────────────────────────────┐
│                                 │
│  "向东3个街区，向北4个街区"      │
│                                 │
│        │ 4                      │
│        │                        │
│        │         ● 目标点       │
│        │       ╱                │
│        │     ╱                  │
│        │   ╱                    │
│  ──────┼──────── 3              │
│        │                        │
│  每个坐标需要独立量化+标准化    │
└─────────────────────────────────┘

极坐标 (PolarQuant):
┌─────────────────────────────────┐
│                                 │
│  "总计5个街区，方向37度角"       │
│                                 │
│        │                        │
│        │   ╱ r=5                │
│        │ ╱ θ=37°               │
│        ●╱                       │
│  ──────┼────────                │
│                                 │
│  半径(强度) + 角度(方向) 分离   │
│  角度分布集中 → 高效量化        │
└─────────────────────────────────┘
```

### 1.3 为什么极坐标更好？

```
┌─────────────────────────────────────────────────────────────┐
│              极坐标的优势                                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  角度模式已知且高度集中:                                     │
│                                                             │
│  笛卡尔:  需要"方形网格"，边界需持续调整                    │
│  ┌─────┬─────┬─────┐                                      │
│  │     │     │     │  ← 每个格子需要量化常数               │
│  ├─────┼─────┼─────┤                                      │
│  │     │  ●  │     │                                      │
│  ├─────┼─────┼─────┤                                      │
│  │     │     │     │                                      │
│  └─────┴─────┴─────┘                                      │
│                                                             │
│  极坐标:  映射到固定"圆形网格"，无需调整                    │
│           ╱ ─ ─ ╲                                          │
│         ╱    │    ╲                                        │
│        │  ───●───  │   ← 角度均匀划分，无需额外常数        │
│         ╲    │    ╱                                        │
│           ╲ ─ ─ ╱                                          │
│                                                             │
│  结果: 无需昂贵的数据标准化步骤!                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Phase 2: QJL — 量化 Johnson-Lindenstrauss 变换

### 2.1 目的

在 PolarQuant 基础上进一步消除量化误差，实现**零内存开销**的精细调整。

### 2.2 核心机制

```
┌─────────────────────────────────────────────────────────────┐
│                    QJL 三步流程                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Step 1: Johnson-Lindenstrauss 降维                         │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  高维向量 (d维)                                      │   │
│  │  ┌────┬────┬────┬────┬─ ─ ─┬────┐                   │   │
│  │  │ x₁ │ x₂ │ x₃ │ x₄ │     │ xd │                   │   │
│  │  └────┴────┴────┴────┴─ ─ ─┴────┘                   │   │
│  │           │                                          │   │
│  │           │  随机投影 (保距变换)                      │   │
│  │           ▼                                          │   │
│  │  低维投影 (保留距离关系)                             │   │
│  │  ┌────┬────┬────┐                                   │   │
│  │  │ y₁ │ y₂ │ y₃ │                                   │   │
│  │  └────┴────┴────┘                                   │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  Step 2: 符号比特化                                         │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  每个投影值 → 仅保留符号 (+1 或 -1)                  │   │
│  │                                                     │   │
│  │  y₁ = 0.35 → +1                                    │   │
│  │  y₂ = -0.12 → -1                                   │   │
│  │  y₃ = 0.78 → +1                                    │   │
│  │                                                     │   │
│  │  结果: 每个维度仅需 1 bit!                          │   │
│  │  内存开销: 零 (符号位已嵌入数据中)                   │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  Step 3: 精度恢复                                           │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  使用特殊估计器在:                                   │   │
│  │    高精度查询 ◀──▶ 低精度简化数据                   │   │
│  │  之间策略性平衡                                     │   │
│  │                                                     │   │
│  │  → 准确计算注意力分数                               │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 关键参数总结

| 参数 | 值 | 作用 | 说明 |
|------|-----|------|------|
| 量化比特数 | 3-4 bit | 控制压缩率 | 3-bit 即可零精度损失 |
| 极坐标维度 | 与原始维度一致 | PolarQuant 转换 | 半径+角度分离 |
| JL 投影维度 | 远小于原始维度 | QJL 降维 | 保距性保证 |
| 内存开销 | 0 bit | QJL 符号编码 | 符号嵌入数据 |

## 总结

TurboQuant 通过以下关键设计实现了 KV 缓存的极致压缩：

1. **PolarQuant**: 极坐标转换消除标准化需求，利用角度集中性高效量化
2. **QJL**: Johnson-Lindenstrauss 变换实现零内存开销的 1-bit 精调
3. **理论保证**: 近最优失真率，有严格的数学证明支撑

核心突破在于从"方形网格"量化转向"圆形网格"量化，消除了传统方法中量化常数的内存开销，在 3-bit 量化下实现零精度损失。
