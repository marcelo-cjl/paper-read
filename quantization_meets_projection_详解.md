## 背景与动机

### 要解决什么问题？

近似最近邻搜索（ANN）是向量数据库的核心，面临"三角困境"：

```
┌─────────────────────────────────────────────────────────────┐
│                    ANN 的三角困境                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│                    准确率                                    │
│                   ╱    ╲                                    │
│                  ╱      ╲                                   │
│                 ╱        ╲                                  │
│                ╱          ╲                                 │
│               ╱            ╲                                │
│              ╱              ╲                               │
│           内存 ──────────── 速度                            │
│                                                             │
│  想要高准确率 → 需要存储更多信息 → 内存大、速度慢          │
│  想要低内存   → 信息被压缩 → 准确率下降                    │
│  想要高速度   → 减少计算 → 准确率或内存受影响              │
│                                                             │
│  传统方法只优化一个维度:                                    │
│  • 量化: 降精度 → 省内存，但维度不变                       │
│  • 投影: 降维度 → 省计算，但精度不变                       │
│                                                             │
│  本文: 同时优化两个维度!                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 为什么量化+投影可以"联姻"？

```
┌─────────────────────────────────────────────────────────────┐
│                互补性分析                                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  量化 (Quantization):                                       │
│  ┌─────────────────────────────────────┐                   │
│  │  作用: 降低每个维度的精度            │                   │
│  │  32-bit → 2-8 bit                   │                   │
│  │                                     │                   │
│  │  优势: 减少存储，加速距离计算        │                   │
│  │  不足: 维度不变，高维仍然慢          │                   │
│  │                                     │                   │
│  │  ┌──┬──┬──┬──┬──┐                  │                   │
│  │  │4b│4b│4b│4b│4b│ ← 维度不变       │                   │
│  │  └──┴──┴──┴──┴──┘                  │                   │
│  └─────────────────────────────────────┘                   │
│                                                             │
│  投影 (Projection):                                         │
│  ┌─────────────────────────────────────┐                   │
│  │  作用: 降低向量维度                  │                   │
│  │  d 维 → d' 维 (d' << d)            │                   │
│  │                                     │                   │
│  │  优势: 减少计算量                    │                   │
│  │  不足: 精度不变，每维仍占32-bit     │                   │
│  │                                     │                   │
│  │  ┌────┬────┬────┐                  │                   │
│  │  │32b │32b │32b │ ← 维度减少       │                   │
│  │  └────┴────┴────┘                  │                   │
│  └─────────────────────────────────────┘                   │
│                                                             │
│  联合 (本文):                                               │
│  ┌─────────────────────────────────────┐                   │
│  │  同时降维度 + 降精度                 │                   │
│  │                                     │                   │
│  │  ┌──┬──┬──┐                        │                   │
│  │  │4b│4b│4b│ ← 维度少、精度低       │                   │
│  │  └──┴──┴──┘                        │                   │
│  │                                     │                   │
│  │  协同效应: 投影保留最判别特征       │                   │
│  │           量化对低维向量更有效       │                   │
│  └─────────────────────────────────────┘                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Phase 1: 投影降维

### 1.1 目的

将高维向量投影到低维空间，保留对最近邻搜索最重要的特征。

### 1.2 投影策略

```
┌─────────────────────────────────────────────────────────────┐
│                   投影阶段                                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  原始向量 x ∈ ℝᵈ (例如 d=1536)                            │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────────────────────────────────┐                  │
│  │  投影矩阵 P ∈ ℝᵈ'×ᵈ               │                  │
│  │                                      │                  │
│  │  x' = P × x                         │                  │
│  │                                      │                  │
│  │  d' = d × 压缩比 (例如 10%-50%)    │                  │
│  │  d=1536, 压缩比=25% → d'=384       │                  │
│  └────┬─────────────────────────────────┘                  │
│       │                                                     │
│       ▼                                                     │
│  投影后向量 x' ∈ ℝᵈ' (d'=384)                            │
│                                                             │
│  关键: P 的选择影响保留哪些距离信息                        │
│  • PCA: 保留方差最大的方向                                 │
│  • 随机投影: JL 保距性保证                                  │
│  • 学习投影: 数据驱动的最优投影                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 投影对距离的影响

```
直觉理解:

高维空间 (d=1536):
┌─────────────────────────────────────────┐
│                                         │
│  A ●────────────── d(A,B) ──────── ● B  │
│                                         │
│  C ●──── d(C,D) ──── ● D               │
│                                         │
│  大部分维度对距离贡献很小               │
│  "维度灾难": 高维中距离趋于相等        │
└─────────────────────────────────────────┘
            │
            │ 投影到 d'=384
            ▼
低维空间 (d'=384):
┌─────────────────────────────────────────┐
│                                         │
│  A'●──────── d'(A',B') ─────── ● B'    │
│                                         │
│  C'●─── d'(C',D') ─── ● D'            │
│                                         │
│  保留了关键的距离关系                   │
│  去除了冗余维度                         │
└─────────────────────────────────────────┘
```

## Phase 2: 量化降精度

### 2.1 目的

对投影后的低维向量进行量化，进一步压缩存储。

### 2.2 量化策略

```
┌─────────────────────────────────────────────────────────────┐
│                   量化阶段                                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  投影后向量 x' ∈ ℝᵈ' (32-bit 每维)                       │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────────────────────────────────┐                  │
│  │  量化函数 Q                          │                  │
│  │                                      │                  │
│  │  标量量化: 每维独立                  │                  │
│  │  x'ᵢ (32-bit) → qᵢ (b-bit)        │                  │
│  │                                      │                  │
│  │  或 乘积量化: 子空间分组             │                  │
│  │  [x'₁..x'ₖ] → 码本索引             │                  │
│  └────┬─────────────────────────────────┘                  │
│       │                                                     │
│       ▼                                                     │
│  量化向量 q (b-bit 每维, b ∈ {2,4,8})                     │
│                                                             │
│  投影+量化的联合压缩效果:                                  │
│  ┌────────────────────────────────────┐                    │
│  │  原始: d × 32 bit = 1536 × 32     │                    │
│  │       = 49,152 bits               │                    │
│  │                                    │                    │
│  │  压缩: d' × b bit = 384 × 4      │                    │
│  │       = 1,536 bits                │                    │
│  │                                    │                    │
│  │  压缩比: 32x!                     │                    │
│  └────────────────────────────────────┘                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 量化对低维向量更有效

```
┌─────────────────────────────────────────────────────────────┐
│           为什么"先投影再量化"比单独量化好？                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  高维量化的问题:                                            │
│  ┌─────────────────────────────────────┐                   │
│  │  1536 维，每维 4-bit                │                   │
│  │  量化误差在各维独立累加              │                   │
│  │  总误差 ∝ d × 单维误差              │                   │
│  │  维度越高，累计误差越大              │                   │
│  └─────────────────────────────────────┘                   │
│                                                             │
│  先降维再量化:                                              │
│  ┌─────────────────────────────────────┐                   │
│  │  投影到 384 维，丢弃不重要的方向    │                   │
│  │  然后对 384 维量化                  │                   │
│  │  总误差 ∝ d' × 单维误差 (d' < d)   │                   │
│  │  误差更集中在重要的方向上            │                   │
│  └─────────────────────────────────────┘                   │
│                                                             │
│  协同效应:                                                  │
│  投影让量化的"预算"集中在最重要的特征上                    │
│  量化节省的空间又可以用来保留更多投影维度                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Phase 3: ANN 搜索

### 3.1 在压缩空间中搜索

```
┌─────────────────────────────────────────────────────────────┐
│                  搜索阶段                                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  查询向量 q_orig ∈ ℝᵈ                                     │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────┐                                              │
│  │  投影    │  q' = P × q_orig                             │
│  └────┬─────┘                                              │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────────────────────────────────┐                  │
│  │  在压缩空间中计算距离                │                  │
│  │                                      │                  │
│  │  对每个数据库向量 xᵢ (已量化):      │                  │
│  │  dist(q', Q(P×xᵢ))                 │                  │
│  │                                      │                  │
│  │  距离计算在低维低精度空间中          │                  │
│  │  → 计算量大幅减少                   │                  │
│  │  → 内存访问大幅减少                 │                  │
│  └────┬─────────────────────────────────┘                  │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────┐                                              │
│  │  排序    │  返回 top-k 最近邻                           │
│  └──────────┘                                              │
│                                                             │
│  可选: 重排序 (re-ranking)                                  │
│  用原始向量对候选集精确计算距离                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 关键参数总结

| 参数 | 范围 | 作用 | 调节建议 |
|------|------|------|----------|
| 量化比特数 b | 2-8 bit | 每维精度 | 4-bit 通常最佳平衡 |
| 投影压缩比 | 10%-50% | 维度保留比例 | 25% 通常足够 |
| 召回目标 | R@1 - R@100 | 准确率要求 | 根据应用调整 |
| 索引结构 | 哈希表/KD树 | 搜索加速 | 根据数据分布选择 |

## 总结

"Quantization Meets Projection" 揭示了量化与投影之间的协同效应：

1. **互补性**: 量化降精度、投影降维度，两者优化不同维度
2. **协同效应**: 投影让量化预算集中在重要特征上，量化节省的空间可保留更多投影维度
3. **实用性**: 在多种数据集上实现 3x 搜索效率提升，同时保持召回率

核心思想：**在向量搜索中，降维和降精度不是二选一的问题，而是可以协同优化的"完美联姻"。**
