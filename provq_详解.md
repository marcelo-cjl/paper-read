## 背景与动机

### 要解决什么问题？

现有 VQ 方法存在一个被忽视的根本问题 —— **过早离散化**：

## Phase 1: 连续学习阶段 (软量化)

### 1.1 目的

让编码器在训练初期自由学习数据的连续表示，不受离散化约束的干扰。

### 1.2 过程

```
┌─────────────────────────────────────────────────────────────┐
│                   软量化阶段                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  输入图像 x                                                 │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────┐                                              │
│  │  编码器  │  → z_e (连续向量)                            │
│  └────┬─────┘                                              │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────────────────────────────────┐                  │
│  │  "软"量化 (量化硬度 τ ≈ 0)           │                  │
│  │                                      │                  │
│  │  z_q = Σ softmax(-‖z_e - eᵢ‖² / τ) × eᵢ              │
│  │        i                             │                  │
│  │                                      │                  │
│  │  τ 很大时: z_q ≈ 所有码本的加权平均  │                  │
│  │  → 几乎等于连续操作                  │                  │
│  │  → 梯度可以自由流动                  │                  │
│  └────┬─────────────────────────────────┘                  │
│       │                                                     │
│       ▼                                                     │
│  ┌──────────┐                                              │
│  │  解码器  │  → 重建 x̂                                   │
│  └──────────┘                                              │
│                                                             │
│  此阶段: 编码器专注于学习数据流形                           │
│          码本开始向数据分布展开                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Phase 2: 渐进退火 (逐步硬化)

### 2.1 目的

将量化硬度从"软"逐步退火到"硬"，实现连续空间到离散空间的平滑过渡。

### 2.2 课程学习策略

```
┌─────────────────────────────────────────────────────────────┐
│                 量化硬度退火曲线                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  量化硬度                                                   │
│  (τ → 0)                                                    │
│     │                                          ┌────────   │
│  硬 │                                     ╱    │ 完全     │
│     │                                ╱         │ 离散     │
│     │                           ╱              │          │
│     │                      ╱                   │          │
│     │                 ╱                        │          │
│     │            ╱  ← 渐进退火                 │          │
│     │       ╱                                  │          │
│  软 │──╱─────────────────────────────────────┘           │
│     └──────────────────────────────────────────────── 训练  │
│         Phase 1        Phase 2         Phase 3     步数    │
│        (纯连续)       (逐步离散)      (完全离散)           │
│                                                             │
│  每一步的量化效果:                                          │
│                                                             │
│  早期 (软):              中期:               晚期 (硬):    │
│  z_q = 加权平均          z_q ≈ 主导码本      z_q = 最近码本│
│  ┌─────────┐            ┌─────────┐         ┌─────────┐   │
│  │ ○ ○ ○   │            │ ○ ◉ ○   │         │ ○ ● ○   │   │
│  │  ╲│╱    │            │   │     │         │   │     │   │
│  │   ●     │            │   ●     │         │   ●     │   │
│  └─────────┘            └─────────┘         └─────────┘   │
│  多个码本贡献             一个为主            仅最近一个    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 渐进退火的好处

```
┌─────────────────────────────────────────────────────────────┐
│               渐进退火 vs 直接量化                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  直接量化 (传统):                                           │
│  ┌────────────────────────────────────────┐                │
│  │  训练步 1:   编码器随机 → 强制离散    │                 │
│  │  训练步 100: 码本仍然混乱              │                │
│  │  训练步 1000: 勉强收敛，码本部分坍缩  │                 │
│  └────────────────────────────────────────┘                │
│                                                             │
│  渐进量化 (ProVQ):                                          │
│  ┌────────────────────────────────────────┐                │
│  │  训练步 1:   编码器自由学习连续表示    │                 │
│  │  训练步 100: 编码器捕捉到数据流形     │                 │
│  │  训练步 500: 量化开始，码本有序展开   │                 │

│  └────────────────────────────────────────┘                │
│                                                             │
│  关键区别:                                                  │
│  "编码器先理解数据，再做量化决策"                           │
```

## 关键参数总结

| 参数 | 作用 | 影响 |
|------|------|------|
| 初始温度 τ₀ | 控制起始软化程度 | 太大→收敛慢，太小→仍有过早离散化 |
| 退火策略 | 温度下降曲线 | 线性/余弦/指数等，影响过渡平滑度 |
| 退火终点 | 何时达到完全离散 | 过早→效果不充分，过晚→训练效率低 |
| 码本大小 | 离散 token 数量 | 与任务复杂度相关 |

## 总结

ProVQ 通过一个简单而有效的课程学习策略解决了 VQ 训练中的"过早离散化"问题：

1. **问题发现**: 首次系统定义"过早离散化"——编码器未成熟时就被迫离散化
2. **渐进退火**: 量化硬度从软到硬平滑过渡，给编码器充分学习时间
3. **码本引导**: 渐进过程有效引导码本朝充分展开的数据流形发展

核心思想：**量化是一个课程，不是一个开关。** 从连续到离散应该是一个平滑的退火过程，而非训练开始时的突然跳变。
